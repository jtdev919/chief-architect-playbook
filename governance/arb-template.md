## 6. Evaluation Criteria

### **6.1 Alignment to Architecture Principles**
- ☐ API‑first  
- ☐ Event‑driven where appropriate  
- ☐ Cloud‑native patterns  
- ☐ Reuse of existing platforms  
- ☐ Security by design  
- ☐ Resiliency & fault tolerance  
- ☐ Observability & operational readiness  

### **6.2 Standards & Governance**
- ☐ API standards (naming, versioning, error handling)  
- ☐ Event schema governance  
- ☐ Data governance (quality, lineage, retention)  
- ☐ Compliance (PCI, HIPAA, SOX, NIST)  
- ☐ Infrastructure standards (IaC, Kubernetes, networking)  

### **6.3 Cross‑Domain Impacts**
- ☐ Upstream/downstream system impacts  
- ☐ Data model changes  
- ☐ Integration dependencies  
- ☐ Security boundaries  
- ☐ Organizational or process impacts  

### **6.4 Operational Readiness**
- ☐ Monitoring & alerting defined  
- ☐ Runbooks / remediation paths  
- ☐ SRE considerations  
- ☐ Deployment strategy (blue/green, canary)  
- ☐ Rollback plan
